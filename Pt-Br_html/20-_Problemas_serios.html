Problemas Sérios

Problemas Sérios

SITUAÇÃO PROBLEMA:

  • Se caso tenha feito algo muito sério, que possa atrapalhar o projeto, você pode resolver com dois comandos:

    • git reflog
    • git cherry-pick

O QUE É GIT REFLOG:

  • Este comando pega todo histórico de ações que você fez no seu repositório, se você fez um reset, rebase, commit. Tudo está gravado, o git não quer perder informações que você faz, por isso que git é muito poderoso.

O QUE DEVO FAZER DEPOIS:

  • Vamos supor que você excluiu seu repositório remoto, e agora?

    • Faz git reflog para pegar o hash.
    • Faz o git cherry-pick "hash que você pegou no reflog".

Pronto!! mas depois que você fazer isso, pode ser que o git, peça para você commitar denovo, ou outro comando, LEIA O COMANDO DO GIT, mas o importante é que com o reflog você consegue corrigir o seu erro.

PRÁTICA:

  • Eu tinha todo meu commit no remoto e local, mas eu fiz uma bestera que apaguei todo meu histórico local e remoto:

Agora fizemos uma besteira, sem querer em voltei para primeiro commit, e sem perceber mandei para o repositório remoto!

COMO RESOLVER?

$ git reflog

>

3218596 (HEAD -> master, origin/master) HEAD@{0}: reset: moving to 321859681d76308922564cc063a5ab8e2b93946f
8b28b77 HEAD@{1}: cherry-pick: commit onze
c8d535e HEAD@{2}: reset: moving to FETCH_HEAD~1
d895553 HEAD@{3}: commit (cherry-pick): commit onze
c8d535e HEAD@{4}: reset: moving to FETCH_HEAD~1
2123492 HEAD@{5}: pull: Fast-forward

Obs: Este comando vai mostrar todo histórico de ação, pode ver que o primeiro da lista eu fiz um reset --hard, para o primeiro commit.

  • Agora pegamos o HASH para onde queremos voltar, vamos voltar para o mais recente:
$ git cherry-pick 8b28b77

Obs: o git vai mostrar um erro, talvez o git peça para resolver conflito, commitar denovo, "git commit --allow-empty", ou talvez precise só:

$ git cherry-pick --continue

e pronto!, porém o git pegar somente as informações, ele não pega todos os commits, vamos ver como ficou:

commit 46adc42a04b3bf660b3162fa8162218eeadd6e6c (HEAD -> master)
Date:   Fri Jul 24 10:22:55 2020 -0300

commit onze

commit 321859681d76308922564cc063a5ab8e2b93946f
Date:   Tue Jul 14 21:43:40 2020 -0300

primeiro commit

 

Observações:

  • Para você pegar novamente cada commit, você teria que fazer um por um , e assim formar o histórico denovo.
  • Garanto que você nunca mais vai querer usar esse comando, porque senão algo de muito grave aconteceu. Só use esse comando por extrema necessidade.
