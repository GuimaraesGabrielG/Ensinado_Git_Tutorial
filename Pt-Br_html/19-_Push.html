Push

O QUE É GIT PUSH?

  • Você fez tudas as suas funcionalidades, adicionou ao git, fez commit, e acha que está tudo correto, então é melhor dar push no seu projeto, explicando melhor, é o comando que você faz para mandar todas as suas alterações para o repositório remeto.
$ git push origin "nome da branch remota"

por exemplo:

$ git push origin master

Obs: Para dar push, o git vai até seu ambiente remoto e compara os histórios de cada ambiente, o local com o remoto, por isso que as vezes, você não irá conseguir dar push se tiver históricos passados diferentes, sempre preste a atenção para aonde você vai querer mandar o push, e sempre faça o git log, para saber estar como você quer.

 

Como fazer um push de uma branch?

  • Como explicado, é preciso ter uma branch para fazer fazer um push, qunado criamos um projeto, ele já vem com uma branch, chamada de master, tambem já ensinamos de como criar um branch localmente, mas não sabemos de como ter esssa branch remota, tudo que você precisa fazer é:
$ git push -u origin "nome da branch"

 

MELHOR FORMA PARA FAZER UM GIT PUSH

  • Existem três passos para se atentar ao fazer um git push:

    • Commit;
    • Pull;
    • Testar o projeto para ver se o pull alterou alguma coisa;
    • Push;
    • Provavelemente o git recusará quando for dar push se você deixar de fazer alguma coisa, então leia a mensagem, se não consigir, busque na internet, ou documentação.

Obs:é sempre importante testar o projeto antes de dar push, porque com estamos trabalhamos com outras pessoas, o push mal feito, pode causar prejuizos ao projeto. se atente também a mensagem com commit, deixe explicativo.

QUANDO USAR O FORCE PUSH ?

  • A verdade que NUNCA se deve usar force para nada, só extrema necessidade, por exemplo eu quero mudar histórico remoto, mas seja conciente em usar esse comando, porque é capaz de fazer coisas erradas.
  • Antes de fazer um force, sempre verifique seu histórico, < git log>, antes e depois de cada comando git.
  • Você irá precisar forçar um push só apenas em duas situações, quando for fazer git rebase e o git --amend, porque esses dois comando muda o histórico local.
