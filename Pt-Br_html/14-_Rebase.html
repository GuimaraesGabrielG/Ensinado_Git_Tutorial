Rebase

Rebase

O QUE É:

  • Rebasing é o processo de mover ou combinar uma sequência de confirmações para uma nova confirmação básica.

    • Fonte: atlassian.com

PARA QUE SERVE?

  • Editar mensagens anteriores do commit;
  • Combinar vários commits em um;
  • Excluir ou reverter commits que não são mais necessários.

É um comando bastante interesante, porque é possível fazer alterações no seu histórico de commits, como por exemplo:

# Commands:
# p, pick <commit> = use commit
# r, reword <commit> = use commit, but edit the commit message
# e, edit <commit> = use commit, but stop for amending
# s, squash <commit> = use commit, but meld into previous commit
# f, fixup <commit> = like "squash", but discard this commit's log message
# x, exec <command> = run command (the rest of the line) using shell
# b, break = stop here (continue rebase later with 'git rebase --continue')
# d, drop <commit> = remove commit
# l, label <label> = label current HEAD with a name
# t, reset <label> = reset HEAD to a label
# m, merge [-C <commit> | -c <commit>] <label> [# <oneline>]
# .       create a merge commit using the original merge commit's
# .       message (or the oneline, if no original merge commit was
# .       specified). Use -c <commit> to reword the commit message.

OBS: O git explica cada comando .

Lembra daquele comando que usamos para juntar branches, é possível fazer de forma interativa (-i), e assim você chamará essa tela com muitas opções de comando:

$ git rebase -i "nome da branch"

O rebase também dá bastante opções, vamos supor que você tenha se arrependido de dar commit, ou algum commit passado tenha ficado ruim, ou deseja renomear, juntar commits. Git rebase pode fazer tudo isso, através desse comando

$ git rebase -i HEAD~"nomero ordenal até onde vocÊ quer ver seu commits"

exemplo:

git rebase -i HEAD~7

Por que 7: porque o git pegar os último 7 que foram adicionados na pilha.
