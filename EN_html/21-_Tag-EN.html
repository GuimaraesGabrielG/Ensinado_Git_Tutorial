Tag

WHAT IS TAG?

  • It is the ability to mark specific points in the history of a repository as being important.

But to place tags correctly it is important to know about SEMANTIC VERSIONING.

WHAT IS SEMANTIC VERSIONING?

  • You assign versions in your project

HOW DOES IT WORK?**

  • We have three spaces to version
  • for example: 0.1.0

  • When we start a project it will start at 1.0.0 !

WHY ?

  • THE MAJOR serves to show the compatibility of the project, for example, we created an initial project just as MVP, and then we have a leaner design with new design and functionality, probably will have very different things from version 1.0.0, then it will be 2.0 or 3.0...
  • MINOR is used to show when it has a new feature in your project, it is dependent on the Major, if the Major changes number, then the minor zeroes and consequently the patch as well.
  • PATCH is used to fix bugs,if you change the minor then reset the patch.

HOW IT WORKS ON GIT

  • To version in git:
$ git tag "version number"

E.g:

$ git tag 1.0.0 #ou
$ git tag v1.0.0

PUSH TAG:

$ git push origin master --tags

Note: consequently you will create a RELEASE on your github.

LIST ALL TAGS:

$ git tag -l

DELETE TAG:

$ git tag -d "tag name"

E.g:

$ git tag -d 1.0.0
